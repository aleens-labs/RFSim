GenAI.mil hosted-site setup required on restricted networks

Why this is needed: api.genai.mil only accepts requests from approved networks. When using the hosted site, run the local relay on this machine so the app can route GenAI.mil traffic through https://127.0.0.1:8788.

Windows
Download Node.js Download Relay Script Download Windows Launcher
1
Install Node.js and download both files

Install Node.js once on this machine, then save genai-proxy.js and start-genai-mil-relay.bat to the same folder.

2
Start the relay

Double-click start-genai-mil-relay.bat.

Manual command: node genai-proxy.js --local-model

3
Create and trust the localhost certificate once

The first run usually creates certs/proxy.crt automatically and then prints the one-time trust command for this workstation.

If the window says OpenSSL is missing, install Git for Windows, open File Explorer in the folder that contains genai-proxy.js, type powershell in the address bar, then run New-Item -ItemType Directory -Force ".\certs" | Out-Null

Then run & "C:\Program Files\Git\usr\bin\openssl.exe" req -x509 -newkey rsa:2048 -keyout ".\certs\proxy.key" -out ".\certs\proxy.crt" -days 3650 -nodes -subj "/CN=localhost" -addext "subjectAltName=IP:127.0.0.1,DNS:localhost"

After that, run node genai-proxy.js --local-model again from the same folder.

4
Return here and retry

Click Refresh Models, then Test Connection.

Local model relay setup required for hosted site

The hosted site needs a small relay on this machine so it can securely reach your local Ollama or LM Studio server through https://127.0.0.1:8788.

Windows
Download Node.js Download Relay Script Download Windows Launcher
1
Install Node.js and your model server

Install Node.js once, then start Ollama, LM Studio, or llama.cpp on this machine.

2
Download and run the local relay

Save genai-proxy.js and start-local-model-relay.bat to the same folder.

Double-click start-local-model-relay.bat.

Manual command: node genai-proxy.js --local-model

3
Create and trust the localhost certificate once

If the relay says OpenSSL is missing, install Git for Windows, open File Explorer in the folder that contains genai-proxy.js, type powershell in the address bar, then run New-Item -ItemType Directory -Force ".\certs" | Out-Null

Then run & "C:\Program Files\Git\usr\bin\openssl.exe" req -x509 -newkey rsa:2048 -keyout ".\certs\proxy.key" -out ".\certs\proxy.crt" -days 3650 -nodes -subj "/CN=localhost" -addext "subjectAltName=IP:127.0.0.1,DNS:localhost"

Then run node genai-proxy.js --local-model again and follow the trust command it prints.

Quick start:
  1. Install Ollama, LM Studio, or llama.cpp and start it locally.
  2. Run node genai-proxy.js --local-model — follow the one-time certificate trust prompt.
  3. Click Detect to find loaded models, then select one from the dropdown.
  4. Click Test Connection.
